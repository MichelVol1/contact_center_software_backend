# **Программное средство автоматизации процессов идентификации и обслуживания клиентов в контакт-центре**

*Описание проекта*  
Современная web-платформа для автоматизации процессов идентификации и обслуживания клиентов в контакт-центре, разработанная с использованием Django (backend) и React (frontend). Система обеспечивает безопасную и эффективную работу агентов с клиентскими данными через единый интерфейс.
*Цель проекта*- Автоматизация процессов идентификации клиентов и сокращение времени обработки обращений  

*Ключевые группы пользователей:*
*1* Операторы и сотрудники контакт-центра ежедневно обрабатывают входящие и исходящие обращения через различные каналы (телефон, чат, email ), нуждаются в эффективных инструментах для идентификации клиентов и ведения истории коммуникаций.  
*2* Руководители и администраторы контакт-центра: контролируют работу персонала, оптимизируют бизнес-процессы, оценивают эффективность, формируют аналитические отчеты, внедряют автоматизацию и контролируют соблюдение стандартов обслуживания.  
*3* Клиенты компаний: конечные потребители, взаимодействующие с контакт-центром для получения услуг, поддержки, консультаций и совершения транзакц

*Основные функции приложения*  
*1* Идентификация и аутентификация клиентов  
*2* Создание и отслеживание тикетов (заявок, обращений)  
*3* Автоматизация типовых сценариев с помощью чат-ботов, IVR и скриптов для операторов  
*4* Мониторинг, аналитика и отчеты по KPI, эффективности работы, ожиданиям и удовлетворенности клиентов  

Ссылки на репозиторий клиента: https://github.com/MichelVol1/contact_center_software_frontend

## **Архитектура**

### C4-модель

![Image](https://github.com/MichelVol1/contact_center_software_backend/blob/main/C4-model.pdf)

Представленные диаграммы следуют методологии C4 (Context, Containers, Components, Code), которая обеспечивает иерархическое описание архитектуры программного обеспечения на разных уровнях детализации.  
Диаграмма контекста (Context)  
Диаграмма контекста показывает систему на самом высоком уровне абстракции, демонстрируя взаимодействие с внешними актерами.  
Пользователи системы:  
• Оператор (Сотрудник контакт-центра) — использует систему для обработки клиентских обращений  
• Руководитель — использует систему для аналитики и планирования работы команды  
Центральная система:  
• Система автоматизации работы контакт-центра — программная платформа, которая помогает в работе операторов и руководителей  
Внешние интеграции:  
• Телефония (Asterisk/FreePBX) — требуется для телефонной связи с клиентами  
• Email-сервисы (WhatsApp, Telegram) — обеспечивают коммуникацию с клиентами через почтовые протоколы  
• Мессенджеры (WhatsApp, Telegram) — предоставляют каналы коммуникации через API мессенджеров  
Диаграмма контейнеров (Containers)  
Диаграмма контейнеров детализирует внутреннюю структуру системы, показывая основные технологические компоненты.  
Клиентская часть (React клиент):  
• React Core Application — ядро веб-приложения  
• UI Components — библиотека пользовательских компонентов интерфейса  
• State Management — управление состоянием клиента  
• API Client — клиент для HTTP/REST запросов  
• WebSocket Client — клиент для real-time коммуникации  
Серверная часть (Django сервер):  
• Django REST Framework — обработка HTTP/REST API  
• WebSocket Handler (Django Channels) — обработка WebSocket соединений и push-уведомлений  
• Business Logic Layer — бизнес-логика системы, обработка данных и взаимодействие с базой данных  
Хранилище данных:  
• База данных (PostgreSQL) — хранит информацию о клиентах, звонках и взаимодействиях  
Внешние сервисы:  
• Email-сервисы (SMTP/IMAP), Телефония (Asterisk/FreePBX), Мессенджеры (WhatsApp, Telegram) через API  
Диаграмма компонентов (Components)  
Диаграмма компонентов раскрывает детальную архитектуру бизнес-слоя системы.  
Уровень представлений (View Layer) через REST API:  
• Auth View/Serializers — аутентификация пользователей  
• Learning View/Serializers — управление обучающими материалами  
• Call View/Serializers — обработка данных о звонках  
• Feedback View/Serializers — работа с отзывами  
• Supervision View/Serializers — управление надзором и KPI  
• Customer View/Serializers — управление данными клиентов  
• Analytics View/Serializers — аналитические данные  
Бизнес-слой (Business Layer):  
Центральный слой обработки бизнес-логики включает:  
• Auth Service — сервис аутентификации  
• Learning Service — управление обучением операторов  
• Call Service — обработка звонков  
• Feedback Service — работа с обратной связью  
• Notification Service — отправка уведомлений  
• Communication Service — мультиканальная коммуникация  
• Customer Service — управление клиентской базой  
• Analytics Service — аналитика и отчеты  
WebSocket компоненты:  
• Chat Consumer, Notification Consumer, Call Consumer, WebSocket Router — обрабатывают real-time коммуникацию  
Уровень моделей (Model Layer):  
• User Model, Learning Model, Interaction Model, Customer Model, Analytics Model — представляют структуру данных  
Внешние интеграции:  
• API Gateway — шлюз для внешних интеграций  
• Django REST Framework — фреймворк для REST API  
• LMS Router — маршрутизация обучающих материалов  
• Middleware Stack — промежуточное ПО  
Диаграмма кода (Code)  
Диаграмма кода показывает структуру классов и их взаимосвязи на уровне реализации. На ней представлены основные сущности:  
Пользователи и роли:  
• User — базовая сущность пользователя  
• Operator — оператор контакт-центра, наследуется от User  
• Supervisor — руководитель группы, управляет KPI и командными метриками  
Клиенты и взаимодействия:
• Customer — информация о клиенте
• Interaction — взаимодействие с клиентом, связывает оператора и клиента
Вспомогательные сущности:  
• Call — информация о звонке  
• Script — скрипт для взаимодействия  
• Template — шаблоны сообщений  
• Feedback — обратная связь от клиента  
• Learning_materials — обучающие материалы для операторов  
Метрики:  
• KPI — ключевые показатели эффективности, определяемые супервизором  
• Team_KPI — командные KPI для групп операторов
